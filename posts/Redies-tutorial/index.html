前言

作为 .NET 开发人员，Redis 是一个必须掌握的技术。Redis 是一个开源的内存数据结构存储系统，可以用作数据库、缓存和消息代理。

什么是 Redis？

Redis（Remote Dictionary Server）是一个基于内存的键值存储系统，具有以下特点：

  • 高性能：数据存储在内存中，读写速度极快
  • 丰富的数据类型：支持字符串、哈希、列表、集合、有序集合等
  • 持久化：支持数据持久化到磁盘
  • 原子性操作：所有操作都是原子性的
  • 支持事务：支持事务操作
  • 发布订阅：支持消息发布/订阅模式

安装 Redis

Windows

  1. 下载 Redis for Windows：访问 https://github.com/microsoftarchive/redis/releases
  2. 解压到指定目录
  3. 运行 redis-server.exe redis.windows.conf 启动服务
  4. 运行 redis-cli.exe 打开客户端
  5. 可选：redis客户端工具：Tiny RDM

tips:

powershell执行请加./ 如 .\redis-server.exe redis.windows.conf

redis.windows.conf 为Windows 专用的 Redis 配置文件，这里启动服务时不加也是默认的redis.windows.conf，此配置文件可修改密码、绑定IP端口、限制最大内存…

Redis 基本数据类型

1. String（字符串）

字符串是 Redis 最基本的数据类型，一个键最大能存储 512MB。

2. Hash（哈希）

哈希是一个键值对集合，适合存储对象。

3. List（列表）

列表是简单的字符串列表，按插入顺序排序。

4. Set（集合）

集合是无序的字符串集合，不允许重复元素。

5. Sorted Set（有序集合）

有序集合在集合的基础上增加了一个分数（score）参数，元素按分数排序。

常见问题

1. 缓存雪崩

大量缓存同时过期，导致请求全部打到数据库。

解决方案：

  • 设置随机过期时间
  • 使用热点数据永不过期
  • 使用互斥锁

2. 缓存穿透

查询不存在的数据，缓存和数据库都没有。

解决方案：

  • 缓存空值
  • 使用布隆过滤器

3. 缓存击穿

热点数据过期瞬间，大量请求同时访问。

解决方案：

  • 使用分布式锁
  • 热点数据永不过期
